# TSLA Options War Room

**一个 AI 驱动的 TSLA 短期期权交易决策系统。**

不是一个简单的 trading bot，也不是一个看图喊单的 prompt。这是一个完整的交易决策基础设施：10 个拟人化精英顾问组成投资委员会，通过对抗式辩论形成方向判断，由本地数据服务提供实时盘面，用交易日志追踪系统准确度。

你跟它聊天就像在跟一个资深交易顾问对话 -- 问得简单它答得简短，问得深入它展开完整分析。系统内部的复杂性对你完全不可见。

## 为什么做这个系统

短期期权交易最难的不是看对方向，而是：

- **在对的位置入场** -- 追高追低是亏钱的主要原因
- **管住手** -- 没有纪律的加仓和扛单是第二大原因
- **快速判断** -- 盘中机会稍纵即逝，不能花 30 分钟做分析

这个系统解决这三个问题：用多 Agent 对抗确保方向判断的质量，用硬编码规则强制执行仓位纪律，用按需数据拉取实现 3 秒内出结果。

## 和普通 Trading Bot 的区别

| | 普通 Trading Bot | War Room |
|---|---|---|
| 方向判断 | 单一模型输出 | 多空对抗辩论后形成共识 |
| 交互方式 | 固定格式输入输出 | 自然语言对话，渐进式展开 |
| 风控 | 简单止损线 | 结构止损 + 合约风险止损 + 时间审查 + 仓位硬规则 |
| 上下文 | 无状态 | 记住持仓、交易日志、历史准确度 |
| 输出风格 | 技术报告 | 像老交易员跟你聊天 |

## 10 个 Agent 角色

每个 Agent 都模拟一个真实的精英从业者，有完整的专业背景、思维风格和工作方法。用户看不到他们 -- 只看到首席顾问（Chief Advisor）汇总后的结论。

| 角色 | 职责 |
|------|------|
| 市场情报员 | 快速读取盘面事实，不做主观判断 |
| 研究协调员 | 发现信息缺口，防止在数据不足时乱出手 |
| 技术结构分析师 | 判断趋势、支撑阻力、当前是好位置还是追位置 |
| 宏观 Beta 顾问 | 判断 QQQ/SPY 是否支持 TSLA 方向 |
| 看空交易主管 | 为 PUT 方向辩护（对抗式辩论） |
| 反转交易员 | 为 CALL 方向辩护（对抗式辩论） |
| 执行规划师 | 把方向判断转化为具体的入场计划和价格区间 |
| 期权结构顾问 | 选 strike、估算成本、评估 theta/gamma/IV 影响 |
| 风控主管 | 执行硬编码仓位规则，不允许情绪交易 |
| 首席顾问 | 汇总所有分析，输出最终结论 |

## 关键设计决策

**方向判断必须对抗** -- 任何涉及 PUT/CALL 的问题，内部必须同时想清楚多方和空方的理由。不允许单边拍脑袋。

**默认给可用答案，不默认给完整报告** -- 用户说"怎么做"不等于要 3000 字报告。先给 15 行可用答案，用户说"展开"才给完整分析。

**仓位规则硬编码** -- 默认 1 张，最多 2 张，第二张只优化成本，一张盈利先卖一张。这些规则不可覆盖。

**交易追踪** -- 入场时记录系统预测，平仓时主动问实际结果，用来衡量系统准确度。

## 快速开始

### 1. Clone 仓库

```bash
cd ~
git clone https://github.com/songzhiyuan98/tsla-options-war-room-skill.git
cd tsla-options-war-room-skill
```

### 2. 安装 Skill

```bash
mkdir -p ~/.claude/skills/tsla-options-war-room
cp SKILL.md ~/.claude/skills/tsla-options-war-room/
cp -r agents/ ~/.claude/skills/tsla-options-war-room/
```

### 3. 配置数据服务

```bash
pip3 install -r requirements.txt
cp .env.example .env
# 编辑 .env，填入 Twelve Data API Key（免费版即可）
# 注册地址：https://twelvedata.com/
```

### 4. 开始使用

```
/tsla-options-war-room TSLA 现在能做期权吗？
```

Skill 触发后会自动拉取最新数据（3 秒），不需要手动启动任何服务。

如果没有配置 API key，也可以手动发截图给 skill 分析。

## 工作原理

**两个位置**：

| 位置 | 内容 | 作用 |
|------|------|------|
| `~/tsla-options-war-room-skill/` | Python 脚本 + 运行时数据 | 数据拉取、快照生成、交易记录 |
| `~/.claude/skills/tsla-options-war-room/` | SKILL.md + agents/ | Claude Code 加载和执行 skill |

Skill 被触发时，去仓库目录读取/刷新数据，然后分析回答。

**数据按需获取**：

```
你提问 -> Skill 检查 snapshot 是否新鲜
              |
    新鲜（<5min）     过期或不存在
         |                  |
      直接用          跑 fetch_now.py（3秒）
         |                  |
         +------分析+回答----+
```

不需要后台常驻进程，不浪费 API 额度。

## 交易参数

| 参数 | 设定 |
|------|------|
| 标的 | TSLA（参考 QQQ / SPY） |
| 期权周期 | 7-10 DTE |
| 默认仓位 | 1 张核心仓 |
| 最大仓位 | 2 张 |
| 基础止盈 | 20%-35% |
| 强趋势止盈 | 40%-70%（TSLA+QQQ 共振时） |
| 结构止损 | thesis 被价格结构否定 |
| 合约止损 | drawdown 25%-35% 且无改善 |
| 时间规则 | 2-3 天无展开必须重审 |

## 注意事项

- 本系统仅供学习和研究用途，不构成投资建议
- 期权交易有重大亏损风险，请在充分了解风险后谨慎操作
- 系统的分析基于技术面数据，不考虑基本面和突发事件
- 建议在模拟账户中测试后再用于实盘
- Twelve Data 免费版有请求限制（800/天），正常使用足够
